# 実データスナップショット運用

日常利用用データは、公式時刻表を参照して `scripts/update-real-data.mjs` でローカルJSONへ
正規化する。アプリ実行時は外部サイトへアクセスせず、生成済みJSONだけを読む。

```powershell
python scripts/extract-bus-pdf-arrivals.py
node scripts/update-real-data.mjs
node scripts/validate-data.mjs
```

## 現在の出典

- 京急: 京急公式「羽田空港第3ターミナル駅 京急蒲田方面」駅時刻表
- 東京モノレール: 東京モノレール公式から案内される時刻表・停車駅一覧
- 京成: 京成公式駅ページから案内される京成本線・成田スカイアクセス線駅時刻表、およびスカイライナー公式時刻表
- JR: JR東日本「空港第2ビル駅」成田・千葉・東京方面 / 成田エクスプレス時刻表
- エアポートバス東京・成田: 公式PDF「成田空港第2ターミナル6番→東京駅方面」時刻表

各ファイルの `metadata.sources.{sourceId}.sourceUrl` に、更新時に照合した公式URLを記録する。
ダイヤ改正後は更新スクリプトを実行し、便数・始発・終発・主要時間帯を公式表示と目視照合してから
JSONを公開する。公式HTML構造が変わった場合は、推測で補完せず更新を停止する。

## 参考地点時刻の取得方針

`arrivalInfo`は標準所要時間の単純加算を原則とせず、次の優先順位で生成する。

1. 列車詳細・列車別停車駅ページから、同一列車の参考地点時刻を取得する。
2. 参考地点の駅時刻表から、列車番号、種別、行先、経路、発車時刻などを使って同一列車を照合する。
3. 安全に照合できない場合は`arrivalInfo`を出力しない。

スカイライナー、成田エクスプレスなど`serviceNumber`を持つ列車では、番号一致を最優先の照合条件とする。
番号がない列車は、種別・行先・経路・時刻・停車駅順を組み合わせて照合する。別列車へ誤照合する
可能性がある場合は、実時刻として公開しない。

スカイライナーは公式専用時刻表の号数と空港第2ビル発時刻を照合し、日暮里の時刻を
`station-timetable / matched`として投入する。成田エクスプレスは`probable`を許可しない。

現在、東京モノレール、京急、京成、JRは、出発駅時刻表から辿れる列車別停車駅・列車詳細ページを
使用して参考地点時刻を取得する。参考地点側の駅時刻表を使う曖昧照合より確度が高いため、
取得できる路線では列車詳細を優先する。

スカイライナーは公式専用時刻表から全便の`serviceNumber`を確定し、号数と発時刻が一致する便だけ
`arrivalInfo`を出力する。成田エクスプレスも`serviceNumber`を必須とする。

エアポートバス東京・成田は、Source名を維持しつつ、利用者向けの種別表示を`バス`とする。
公式PDFの列位置を使って、成田空港第2ターミナル発と東京駅の便別到着予定時刻を対応付ける。
PDFから抽出した値は`data/tyo-nrt-arrivals.json`へ保存し、`manual / manual`として投入する。

成田第2の京成系統は、取得元に存在する全列車をJSONへ投入する。初期表示対象は
`defaultFilter`で制御し、データ取得段階では快速・普通・通勤特急・ライナー等を除外しない。
参考地点は青砥を表示対象から外し、日暮里、押上、日本橋を列車経路に応じて出力する。
日本橋は列車詳細から取得できた便だけ`arrivalInfo`へ入れる。

成田エクスプレスは列車詳細で同一号数の複数終着駅を確認できた場合、`destination`を
`新宿・大船`のように結合する。
東京の参考地点時刻は、同一号数内で複数の東京発時刻がある場合、最も早い時刻を採用する。

## 現在の制約

- `weekday` は平日、`holiday` は土休日ページを使用する。京急は公式の「休日」ページを採用しているため、
  土曜と休日に差がある改正では `saturday` の追加が必要。
- `arrivalInfo`は列車別公式ページから取得する。固定所要時間による推定値は使用しない。
- スカイライナーの `serviceNumber` は公式専用時刻表の「号」列から投入する。
- JR普通・快速の行先は駅時刻表上の方面表示を優先する。成田エクスプレスは列車詳細から終着駅を反映する。
- バスの便別到着予定時刻は公式PDFに明記された値だけを使用し、固定加算は使用しない。
